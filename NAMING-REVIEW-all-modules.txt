NAMING PRINCIPLES
================================================================================

1. **Common Prefix Rule**: Related modules should share a common domain prefix
   - Example: pet_feedback, pet_thoughts, pet_engine (all "pet_" prefix)

2. **Specificity Rule**: Names should be specific, not generic
   - BAD: "feedback" (feedback for what?)
   - GOOD: "pet_feedback" (pet behavior feedback)

3. **Function Clarity**: Name should describe what the module DOES
   - Pattern: {domain}_{function}
   - Example: pet_feedback (pet domain, feedback function)

4. **Layer Prefix**: c{N}_ prefix indicates architectural layer
   - c1_ = Foundation (models, infrastructure)
   - c2_ = Services (business logic)
   - c3_ = Applications (UI, CLI)
